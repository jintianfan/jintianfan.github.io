发表于 2017-12-12 | 阅读次数:

现象描述

kafka在磁盘写满情况下，会抛出异常导致broker进程crash。

异常分析

ReplicaFetcherThread中抛出FATAL的代码

Runtime.getRuntime.halt(1) 强制退出当前进程，从而导致kafka broker进程崩溃。

System.exit(status)和Runtime.getRuntime().halt(status)区别

针对System.exit(status)解释如下：

Terminates the currently running Java Virtual Machine. The
argument serves as a status code; by convention, a nonzero status
code indicates abnormal termination

针对Runtime.getRuntime().halt(status)解释如下：

Forcibly terminates the currently running Java virtual machine. This
method never returns normally.
